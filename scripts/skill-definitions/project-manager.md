## 📚 核心框架

### 项目管理五大过程组

```
PMBOK五大过程组:
启动 → 规划 → 执行 → 监控 → 收尾

启动Initiating:
├── 项目章程: 项目目的/成功标准/里程碑/项目经理权限
├── 干系人识别: 谁影响项目/谁被项目影响
└── 输出: 明确范围边界 + 授权项目经理

规划Planning:
├── 范围管理: WBS工作分解结构/范围基线
├── 时间管理: 活动排序/关键路径CPM/甘特图
├── 成本管理: 资源估算/成本估算/预算制定
├── 质量管理: 质量标准/质量指标/QA计划
├── 风险管理: 风险识别/分析/应对规划
├── 沟通管理: 谁在何时收到什么信息
└── 采购管理: 外包决策/合同类型

执行Executing:
├── 团队建设和管理
├── 质量保证QA
├── 信息分发
└── 干系人参与管理

监控Monitoring:
├── 绩效监控: 挣值管理EVM
├── 整体变更控制: CCB变更控制委员会
├── 偏差分析和纠正措施
└── 风险监控

收尾Closing:
├── 可交付成果验收
├── 经验教训总结
├── 组织过程资产更新
└── 项目后评审
```

### 挣值管理EVM

```
挣值管理三要素:
├── PV计划值: 计划完成工作的预算成本
├── EV挣值: 实际完成工作的预算成本
└── AC实际成本: 实际完成工作的实际成本

绩效指标:
├── SPI进度绩效指数 = EV / PV
│   ├── >1: 进度提前
│   ├── =1: 进度符合
│   └── <1: 进度落后
└── CPI成本绩效指数 = EV / AC
    ├── >1: 成本节约
    ├── =1: 成本符合
    └── <1: 成本超支

预测指标:
├── EAC完工估算 = BAC / CPI
├── ETC完工尚需估算 = EAC - AC
└── VAC完工偏差 = BAC - EAC
```

### 敏捷Scrum框架

```
Scrum三个角色:
├── Product Owner产品负责人: 管理Product Backlog/确定优先级/验收标准
├── Scrum Master敏捷教练: 保护团队/移除障碍/确保流程正确
└── Development Team开发团队: 自组织/跨职能/3-9人

Scrum五个事件:
├── Sprint迭代: 2-4周固定周期
├── Sprint Planning规划会: 选择Backlog项/制定计划
├── Daily Scrum每日站会: 15分钟/昨天做了什么/今天做什么/有什么障碍
├── Sprint Review评审会: 演示成果/收集反馈
└── Sprint Retrospective回顾会: 改进流程/团队反思

Scrum三个工件:
├── Product Backlog产品待办列表: 优先级排序的需求列表
├── Sprint Backlog迭代待办列表: 本次迭代要完成的任务
└── Increment产品增量: 可交付的产品功能

Scrum价值观:
├── 承诺Commitment: 承诺完成Sprint目标
├── 专注Focus: 专注于Sprint工作
├── 开放Openness: 对工作和挑战开放
├── 尊重Respect: 尊重团队成员
└── 勇气Courage: 有勇气做正确的事
```

### 风险管理

```
风险管理六步法:
规划风险管理 → 识别风险 → 定性分析 → 定量分析 → 规划应对 → 监控风险

风险识别:
├── 头脑风暴
├── 德尔菲技术
├── 核对单分析
├── SWOT分析
└── 假设日志分析

风险定性分析:
├── 概率影响矩阵
├── 风险优先级排序
└── 风险分类

风险定量分析:
├── 蒙特卡洛模拟
├── 决策树分析
├── 敏感性分析
└── 期望货币值EMV

风险应对策略:
├── 消极风险: 规避/转移/减轻/接受
└── 积极风险: 开拓/分享/提高/接受
```

### 变更管理

```
变更管理流程:
变更请求 → 影响分析 → CCB审批 → 实施变更 → 变更验证

变更请求类型:
├── 纠正措施: 纠正偏差
├── 预防措施: 预防风险
├── 缺陷补救: 修复缺陷
└── 更新: 文档/计划更新

影响分析维度:
├── 范围影响
├── 进度影响
├── 成本影响
├── 质量影响
├── 风险影响
└── 干系人影响

变更控制委员会CCB:
├── 组成: 项目经理/关键干系人/技术专家
├── 职责: 审批或拒绝变更请求
└── 原则: 所有变更必须经过CCB
```

### 项目沟通

```
沟通模型:
发送者 → 编码 → 信息 → 解码 → 接收者 → 反馈

沟通方式:
├── 正式沟通: 会议/报告/邮件
├── 非正式沟通: 聊天/电话/即时通讯
├── 书面沟通: 文档/邮件/报告
└── 口头沟通: 会议/电话/面对面

沟通频率:
├── 日报: 每日站会/进度报告
├── 周报: 周例会/周进度报告
├── 月报: 月度总结/里程碑报告
└── 里程碑报告: 阶段性成果汇报

干系人管理:
├── 识别干系人
├── 分析干系人: 权力/利益矩阵
├── 制定沟通策略
└── 管理干系人期望
```
