## 📚 核心框架

### 设计思维五步法

```
设计思维流程:
同理心 → 定义 → 创意 → 原型 → 测试

1. 同理心Empathize:
├── 用户访谈: 开放式问题/深度对话
├── 观察法: 自然环境观察用户行为
├── 沉浸体验: 自己成为用户
└── 输出: 用户画像/用户旅程地图

2. 定义Define:
├── 提炼核心问题
├── HMW问题框架: "我们可能如何..."
└── POV声明: [用户]需要[做什么]因为[原因]

3. 创意Ideate:
├── 发散: 不评判/追求数量/鼓励疯狂想法
├── 收敛: 投票筛选/可行性评估
└── 方法: 头脑风暴/SCAMPER/类比迁移

4. 原型Prototype:
├── 低保真: 纸原型/草图
├── 中保真: 线框图
└── 高保真: 可交互原型

5. 测试Test:
├── 用户测试: 真实用户使用
├── 观察反馈: 观察行为
└── 迭代优化: 根据反馈改进
```

### 双钻模型

```
双钻模型:
第一个钻石(发现正确的问题):
├── Discover发现: 用户研究/竞品分析/市场调研
└── Define定义: 聚焦核心问题和机会点

第二个钻石(找到正确的解):
├── Develop开发: 头脑风暴/草图/快速原型
└── Deliver交付: 高保真设计/开发交付/验证

关键: 确保解决对的问题而不只是把问题解决好
```

### 用户研究方法

```
用户研究矩阵:
┌──────────────┬─────────────────┬─────────────────┐
│              │ 定性Qualitative │ 定量Quantitative│
├──────────────┼─────────────────┼─────────────────┤
│ 行为Behavior │ 可用性测试      │ 点击热力图      │
│              │ 情境访谈        │ 漏斗分析        │
│              │ A/B测试         │ 眼动追踪        │
├──────────────┼─────────────────┼─────────────────┤
│ 态度Attitude │ 用户访谈        │ 问卷调查        │
│              │ 焦点小组        │ NPS评分         │
│              │ 日记研究        │ 满意度评分      │
└──────────────┴─────────────────┴─────────────────┘

用户画像Persona:
├── 基本信息: 姓名/年龄/职业/收入
├── 目标需求: 用户目标/核心需求
├── 行为特征: 使用习惯/偏好
├── 痛点: 当前遇到的问题
└── 场景: 典型使用场景

用户旅程地图:
├── 阶段: 认知→考虑→购买→使用→忠诚
├── 行为: 每个阶段用户做什么
├── 情绪: 每个阶段用户感受
├── 痛点: 每个阶段遇到的问题
└── 机会: 每个阶段的改进机会
```

### UI设计原则

```
视觉层次:
├── 大小Size: 最重要的最大
├── 颜色Color: 对比色突出关键元素
├── 间距Space: 更多空间=更重要
├── 粗细Weight: 粗体=重要
└── 位置Position: 左上角最先被看到

格式塔原则:
├── 接近性: 距离近的元素被视为一组
├── 相似性: 相似的元素被视为一组
├── 连续性: 视觉沿线条流动
├── 闭合性: 大脑自动填充缺失部分
└── 图底关系: 区分主体和背景

排版原则:
├── 字体选择: 不超过2种字体
├── 字号层级: 标题/正文/辅助文字
├── 行高: 1.5-1.8倍字号
├── 对齐: 左对齐/居中对齐
└── 对比: 标题与正文对比

色彩原则:
├── 60-30-10法则: 主色60%/辅助色30%/点缀色10%
├── 色彩对比: 确保可读性
├── 色彩语义: 红色警告/绿色成功/蓝色信息
└── 无障碍设计: 考虑色盲用户
```

### 设计系统

```
设计系统组成:
├── 设计原则: 设计理念/价值观
├── 设计Token: 颜色/字体/间距/圆角
├── 组件库: 基础组件/业务组件
├── 模式库: 常见设计模式
└── 指南文档: 使用规范/最佳实践

设计Token:
├── 颜色:
│   ├── 主色: Primary/Secondary
│   ├── 语义色: Success/Warning/Error/Info
│   └── 中性色: Text/Background/Border
├── 字体:
│   ├── 字体家族: 标题字体/正文字体
│   ├── 字号: H1-H6/Body/Small
│   └── 字重: Regular/Medium/Bold
├── 间距: 4px基准/8px/12px/16px/24px/32px
└── 圆角: Small/Medium/Large/Full

组件库:
├── 基础组件:
│   ├── Button按钮
│   ├── Input输入框
│   ├── Select选择器
│   ├── Checkbox复选框
│   └── Radio单选框
└── 业务组件:
    ├── Card卡片
    ├── Table表格
    ├── Form表单
    └── Modal弹窗
```

### 可用性测试

```
测试类型:
├── 形成性测试: 设计过程中发现可用性问题
├── 总结性测试: 设计完成后评估可用性
└── A/B测试: 对比不同方案效果

测试流程:
├── 准备:
│   ├── 定义测试目标
│   ├── 编写测试任务
│   ├── 招募测试用户(5-8人)
│   └── 准备测试环境
├── 执行:
│   ├── 欢迎与说明
│   ├── 前置访谈
│   ├── 任务执行(有声思维)
│   └── 后置访谈
└── 分析:
    ├── 整理问题清单
    ├── 优先级排序
    └── 提出改进建议

测试任务设计:
├── 具体场景: "假设你想购买..."
├── 明确目标: "找到并购买..."
├── 不给提示: 不告诉用户如何操作
└── 观察行为: 关注用户说什么和做什么
```

### 响应式设计

```
断点设计:
├── Mobile: < 768px
├── Tablet: 768px - 1024px
├── Desktop: 1024px - 1440px
└── Large Desktop: > 1440px

响应式策略:
├── 移动优先: 从移动端开始设计
├── 流式布局: 使用百分比/弹性单位
├── 媒体查询: 根据断点调整样式
└── 图片适配: 响应式图片/懒加载

移动端设计:
├── 触摸友好: 按钮最小44x44px
├── 简化导航: 汉堡菜单/底部导航
├── 内容优先: 核心内容前置
└── 性能优化: 减少请求/压缩资源
```
