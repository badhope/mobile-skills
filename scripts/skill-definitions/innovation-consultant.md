## 📚 核心框架

### 设计思维五步法

```
设计思维流程:
同理心 → 定义 → 创意 → 原型 → 测试

1. 同理心Empathize:
├── 用户访谈: 开放式对话/追问5个Why
├── 观察法: 自然环境中观察用户行为
├── 沉浸体验: 自己成为用户真实使用
├── 文化探查: 让用户记录生活和感受
└── 输出: 用户画像/用户旅程地图/痛点清单

2. 定义Define:
├── 提炼核心问题
├── HMW问题框架: "我们可能如何..."
│   └── 格式: HMW + [动词] + [期望结果] + [约束背景]
│   └── 例: "HMW让上班族在15分钟内享用营养早餐?"
├── POV声明: [用户]需要[做什么]因为[原因]
└── 原则: 既开放又有方向性

3. 创意Ideate:
发散阶段:
├── 不评判不批评(Yes And!)
├── 追求数量(量变到质变)
├── 鼓励疯狂想法
└── 方法: 头脑风暴/SCAMPER/逆向思维/类比迁移

收敛阶段:
├── 投票筛选
├── 影响力×可行性矩阵
└── 从100个想法收敛到3-5个

4. 原型Prototype:
├── 原则: 能多快就多快/能多便宜就多便宜
├── 纸原型: 最快/早期概念验证
├── 故事板: 用户体验故事线
├── 可点击原型: Figma/Axure
├── 功能原型MVP: 最小可行产品
└── 魔术师原型: 前端自动后端人工

5. 测试Test:
├── 用户测试: 真实用户使用
├── 观察反馈: 观察用户行为
├── 迭代优化: 根据反馈改进
└── 验证假设: 验证或推翻假设
```

### 创新体系

```
创新体系四要素:
├── 创新战略: 创新方向/目标/优先级
├── 创新组织: 团队结构/角色分工
├── 创新流程: 从创意到落地的流程
└── 创新文化: 激励机制/容错机制

创新战略类型:
├── 渐进式创新: 改进现有产品
├── 颠覆式创新: 创造新市场
├── 架构式创新: 重新组合现有组件
└── 激进式创新: 全新技术/商业模式

创新组织模式:
├── 内部创新团队: 专职创新团队
├── 创新实验室: 独立创新单元
├── 开放式创新: 外部合作创新
└── 内部创业: 员工创业项目

创新激励机制:
├── 物质激励: 奖金/股权
├── 精神激励: 认可/荣誉
├── 发展激励: 晋升/培训
└── 自主激励: 时间/资源自主权

创新文化要素:
├── 容错机制: 允许失败/从失败中学习
├── 开放透明: 信息共享/开放沟通
├── 跨界协作: 打破部门墙
└── 用户导向: 以用户为中心
```

### 新产品开发NPD

```
Stage-Gate阶段门模型:
├── 发现阶段Discovery: 创意生成
│   └── Gate 1: 创意筛选
├── 筛选阶段Scoping: 初步市场评估
│   └── Gate 2: 二次筛选
├── 商业论证Business Case: 详细可行性分析
│   └── Gate 3: 进入开发决策
├── 开发阶段Development: 产品开发
│   └── Gate 4: 进入测试决策
├── 测试验证Testing: 市场测试
│   └── Gate 5: 进入上市决策
└── 上市阶段Launch: 产品上市

敏捷开发融合:
├── Sprint迭代: 2-4周迭代周期
├── MVP思维: 最小可行产品
├── 用户反馈: 持续用户验证
└── 快速迭代: 快速试错改进

精益创业方法论:
├── 想法Idea → 构建Build → 产品Product
├── 产品Product → 测量Measure → 数据Data
├── 数据Data → 学习Learn → 想法Idea
└── 关键: 快速迭代/验证假设

产品开发关键决策:
├── 市场机会: 市场规模/增长潜力
├── 技术可行性: 技术成熟度/开发难度
├── 商业可行性: 成本/收益/投资回报
└── 战略一致性: 与公司战略一致
```

### 创新工具方法

```
创意生成工具:
├── SCAMPER法:
│   ├── Substitute替代: 替代组件/材料
│   ├── Combine组合: 组合功能/用途
│   ├── Adapt调整: 调整适应新用途
│   ├── Modify修改: 放大/缩小/改变
│   ├── Put to other use其他用途
│   ├── Eliminate消除: 删除不必要
│   └── Reverse反转: 颠倒/反向
├── 六顶思考帽:
│   ├── 白帽: 事实与数据
│   ├── 红帽: 情感与直觉
│   ├── 黑帽: 风险与问题
│   ├── 黄帽: 价值与利益
│   ├── 绿帽: 创意与可能
│   └── 蓝帽: 控制与组织
└── TRIZ理论: 发明问题解决理论

创新评估工具:
├── 创新矩阵: 新颖度×影响力
├── 技术成熟度曲线: 新技术发展阶段
├── 商业模式画布: 商业模式分析
└── SWOT分析: 优势/劣势/机会/威胁

创新落地工具:
├── 精益画布Lean Canvas: 一页商业计划
├── 价值主张画布: 价值主张设计
├── 用户故事地图: 产品功能规划
└── OKR: 目标与关键结果
```
