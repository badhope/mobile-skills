## 📚 核心框架

### ADDIE课程开发模型

```
ADDIE五步法:
Analysis分析 → Design设计 → Develop开发 → Implement实施 → Evaluate评估

Analysis分析:
├── 学习者分析: 年龄/背景/现有知识/学习动机
├── 需求分析: 绩效差距 = 期望表现 - 当前表现
├── 约束条件: 时间/预算/场地/技术
└── 输出: 培训需求分析报告 + 学习目标

Design设计:
├── 教学策略: 讲授/案例/角色扮演/模拟/游戏化
├── 课程结构: 开场→内容模块→练习→总结
├── 时间分配: 每模块时间安排
├── 评估方式: 如何知道学员学会了
└── 输出: 课程设计蓝图 + 详细大纲

Develop开发:
├── 教学材料: PPT/学员手册/练习题/案例
├── 配套资源: 课前阅读/课后作业/工具模板
├── 试讲评审: 内部试讲/收集反馈/修改完善
└── 输出: 完整课程包

Implement实施:
├── 培训前: 场地设备/学员通知/物料准备
├── 培训中: 按计划执行/灵活应对
└── 培训后: 发放资料/行动任务/社群建立

Evaluate评估:
├── 反应层: 学员满意度
├── 学习层: 知识技能掌握
├── 行为层: 工作行为改变
└── 结果层: 业务绩效提升
```

### 成人学习原则

```
成人学习5大特征:
├── 自我导向: 参与决定学什么怎么学
├── 经验为基础: 利用已有经验作为学习资源
├── 问题中心: 想解决问题而非积累知识
├── 学习意愿与社会角色相关: 学"对我有用"的东西
└── 内外动机结合: 成长兴趣(内)+晋升认证(外)

70-20-10学习法则:
├── 70%实践经验: 做中学/项目/挑战性任务
├── 20%社交学习: 导师指导/反馈/观察他人
└── 10%正式学习: 培训课程/阅读/认证

学习目标ABCD法:
├── Audience: 谁
├── Behavior: 做什么
├── Condition: 在什么条件下
└── Degree: 做到什么程度
```

### 授课技巧

```
开场技巧:
├── 引起注意: 提问/故事/数据/演示
├── 建立关联: 与学员经验/工作相关
├── 明确目标: 学完能做什么
└── 激发兴趣: 为什么这很重要

内容呈现:
├── 结构清晰: 总-分-总/时间顺序/逻辑顺序
├── 重点突出: 3-5个核心要点
├── 举例说明: 具体案例/数据支持
├── 视觉辅助: PPT/视频/实物演示
└── 节奏控制: 重点慢讲/过渡快讲

互动技巧:
├── 提问: 开放式/封闭式/反问
├── 讨论: 小组讨论/头脑风暴
├── 练习: 角色扮演/案例分析/实操
├── 游戏: 破冰游戏/教学游戏
└── 反馈: 正向反馈/建设性反馈

控场技巧:
├── 时间管理: 严格控制时间/预留缓冲
├── 突发处理: 设备故障/学员挑战/冷场
├── 氛围营造: 热情/幽默/专业
└── 结尾总结: 回顾要点/行动号召
```

### 培训评估

```
Kirkpatrick四级评估:
├── Level 1反应层: 学员满意度/培训体验
│   └── 方法: 满意度问卷/访谈
├── Level 2学习层: 知识技能掌握程度
│   └── 方法: 测试/实操考核/角色扮演
├── Level 3行为层: 工作中行为改变
│   └── 方法: 观察/360度反馈/上级评价
└── Level 4结果层: 业务绩效提升
    └── 方法: KPI数据/业务指标对比

评估时机:
├── 反应层: 培训结束时
├── 学习层: 培训结束时
├── 行为层: 培训后1-3个月
└── 结果层: 培训后3-6个月

ROI计算:
ROI = (收益 - 成本) / 成本 × 100%
├── 收益: 生产力提升/错误减少/员工保留
└── 成本: 培训费用/时间成本/机会成本
```

### 课程设计模板

```
课程大纲结构:
├── 课程名称
├── 课程目标
├── 学员对象
├── 课程时长
├── 课程内容
│   ├── 模块1: 主题/目标/内容/方法/时间
│   ├── 模块2: 主题/目标/内容/方法/时间
│   └── 模块3: 主题/目标/内容/方法/时间
└── 评估方式

PPT设计原则:
├── 一页一主题
├── 文字少而精(每页<6行)
├── 图表胜过文字
├── 字体大小: 标题>32pt/正文>24pt
├── 配色: 主色+辅助色(不超过3种)
└── 动画: 简洁/适度/不干扰

学员手册:
├── 课程介绍
├── 学习目标
├── 核心内容
├── 练习活动
├── 参考资料
└── 行动计划
```

### 常见培训方法

```
讲授法:
├── 优点: 信息量大/效率高/成本低
├── 缺点: 互动少/被动学习
└── 适用: 知识传授/概念讲解

案例分析法:
├── 优点: 联系实际/激发思考
├── 缺点: 案例质量要求高/耗时
└── 适用: 问题解决/决策能力

角色扮演:
├── 优点: 实践性强/体验深刻
├── 缺点: 学员抗拒/需要引导
└── 适用: 沟通技巧/销售技能

小组讨论:
├── 优点: 互动性强/经验分享
├── 缺点: 可能跑题/需要控场
└── 适用: 问题探讨/创意生成

游戏化学习:
├── 优点: 趣味性强/参与度高
├── 缺点: 设计复杂/可能偏离目标
└── 适用: 团队建设/技能训练
```
