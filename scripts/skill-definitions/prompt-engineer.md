## 📚 核心框架

### Prompt设计原则

```
核心原则:
├── 明确性Specificity: 具体指令优于模糊描述
│   ├── ❌ "帮我分析一下这个数据"
│   └── ✅ "用Python pandas读取sales_2024.csv，按月份统计销售额，绘制折线图"
├── 结构化Structure: 使用Markdown/XML/JSON组织
│   ├── ## Role: 角色定义
│   ├── ## Task: 任务描述
│   ├── ## Constraints: 约束条件
│   ├── ## Output Format: 输出格式
│   └── ## Examples: 示例
├── 上下文Context: 提供足够背景信息和示例
│   └── Few-Shot Learning: 提供输入输出示例
└── 约束Constraints: 明确限制输出长度/格式/风格
    ├── 必须做什么
    └── 不能做什么
```

### 高级Prompt技术

```
思维链Chain-of-Thought:
├── Zero-shot CoT:
│   └── "Let's think step by step"
├── Manual CoT:
│   └── 手动提供推理过程示例
├── Auto-CoT:
│   └── 自动生成多样化示例
└── Tree-of-Thought ToT:
    └── 探索多条推理路径/回溯选择最优解

Self-Consistency自洽性:
├── 同一问题多次采样(temperature>0)
├── 取多数投票(Majority Voting)
└── 提高准确性

Few-Shot Learning:
├── Zero-Shot: 无示例
├── One-Shot: 一个示例
├── Few-Shot: 多个示例
└── 最佳实践: 3-5个示例

Prompt Chaining:
├── 将复杂任务分解为多个步骤
├── 每步一个Prompt
├── 前一步输出作为后一步输入
└── 适合复杂推理任务
```

### System Prompt设计

```
System Prompt结构:
├── 角色定义Role:
│   └── "你是一个专业的[角色]"
├── 能力描述Capabilities:
│   └── "你可以..."
├── 行为约束Constraints:
│   ├── "你必须..."
│   └── "你不能..."
├── 输出规范Output Format:
│   └── "输出格式为..."
└── 示例Examples:
    └── 提供输入输出示例

角色设定技巧:
├── 专业角色: "你是一个资深[职业]"
├── 特定视角: "从[角度]分析"
├── 风格设定: "用[风格]回答"
└── 受众定位: "面向[受众]解释"

行为约束:
├── 格式约束: "输出JSON格式"
├── 长度约束: "回答不超过200字"
├── 风格约束: "使用专业术语"
└── 安全约束: "不涉及敏感话题"
```

### Prompt优化方法

```
优化流程:
├── 定义目标: 明确期望输出
├── 初始Prompt: 编写基础版本
├── 测试评估: 多样化测试
├── 问题分析: 识别失败模式
├── 迭代优化: 针对性改进
└── A/B测试: 对比不同版本

评估指标:
├── 准确性: 输出是否正确
├── 一致性: 多次输出是否稳定
├── 相关性: 是否回答问题
├── 完整性: 是否覆盖所有要求
└── 格式性: 是否符合格式要求

常见问题:
├── 输出不一致: 增加约束/降低temperature
├── 格式错误: 提供明确格式示例
├── 内容偏离: 强化上下文/约束
├── 过度冗长: 增加长度限制
└── 理解偏差: 提供更多示例

优化技巧:
├── 添加示例: Few-Shot Learning
├── 分步引导: Chain-of-Thought
├── 明确格式: 结构化输出
├── 角色强化: 明确角色定位
└── 约束细化: 具体化约束条件
```

