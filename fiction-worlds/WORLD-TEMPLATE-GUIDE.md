## 1. 概述

### 1.1 什么是互动小说世界模板？

互动小说世界模板是一个完整的、自包含的虚构世界设定文档。用户通过让 AI 读取该文档，即可进入该世界进行沉浸式互动体验。

### 1.2 核心设计理念

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         互动小说设计理念                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   📖 完整性 - 一个文档包含世界的所有必要信息                                   │
│   🎮 可玩性 - 有明确的游戏机制和互动方式                                       │
│   🌍 沉浸感 - 丰富的世界观和细节设定                                          │
│   🔓 开放性 - 用户有自由探索和选择的空间                                       │
│   ⚖️ 平衡性 - 规则约束与创意发挥的平衡                                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.3 与现有 Agent 的区别

| 维度 | 角色 Agent | 互动小说世界模板 |
|:-----|:-----------|:-----------------|
| **核心** | 扮演一个角色 | 构建一个完整世界 |
| **交互** | 角色对话为主 | 世界互动 + 角色扮演 |
| **系统** | 无/简单 | 完整的游戏系统 |
| **内容** | 角色设定 | 世界观 + 规则 + NPC + 事件 |
| **目标** | 情感陪伴 | 沉浸式体验 |

### 2.2 各章节详细说明

#### 2.2.1 世界概述

**目的**: 让 AI 快速理解世界的核心概念

**必填内容**:
- 世界名称和类型
- 核心主题（一句话概括）
- 内容分级（全年龄/青少年/成人）

**示例**:
```markdown
## 1. 世界概述

### 1.1 世界简介
这是一个修仙世界，玩家将扮演一名初入修仙界的凡人，
通过修炼、历练、机缘，最终追求长生大道。

### 1.2 核心主题
凡人修仙，逆天改命

### 1.3 内容分级
青少年向 (12+) - 包含战斗描写，无成人内容
```

#### 2.2.2 世界观设定

**目的**: 构建沉浸式的世界背景

**建议内容**:
- 历史背景：重要历史事件、纪元划分
- 地理环境：主要区域、地标、气候
- 势力阵营：主要势力、关系、目标
- 文化风俗：习俗、禁忌、价值观

**深度建议**:
- 精简型：每个部分 100-200 字
- 标准型：每个部分 300-500 字
- 详尽型：每个部分 500+ 字，可单独成章

#### 2.2.3 核心规则系统

**目的**: 定义游戏的数值框架

**必须明确定义**:

1. **属性系统**
```markdown
### 3.1 属性系统

| 属性 | 说明 | 影响 |
|:-----|:-----|:-----|
| 力量 | 肉体力量 | 物理伤害、负重 |
| 敏捷 | 灵活程度 | 闪避、速度 |
| 体质 | 身体强度 | 生命值、抗性 |
| 智力 | 思维能力 | 法术伤害、学习 |
| 感知 | 洞察力 | 侦查、先攻 |
| 魅力 | 社交能力 | 说服、交易 |

初始属性分配：玩家有 72 点可分配，单项最低 6，最高 18。
```

2. **技能系统**
```markdown
### 3.2 技能系统

技能等级：新手(1) → 熟练(2) → 精通(3) → 大师(4) → 宗师(5)

| 技能 | 关联属性 | 说明 |
|:-----|:---------|:-----|
| 剑术 | 力量/敏捷 | 使用剑类武器 |
| 炼丹 | 智力 | 炼制丹药 |
| 驭兽 | 魅力/感知 | 驾驭灵兽 |
```

3. **物品/装备系统**
```markdown
### 3.3 物品/装备系统

物品稀有度：
- 普通 (白色) - 常见物品
- 精良 (绿色) - 略有特殊
- 稀有 (蓝色) - 较为珍贵
- 史诗 (紫色) - 极其珍贵
- 传说 (金色) - 传世之宝

装备槽位：武器、防具、饰品x2、法宝
```

4. **状态/资源系统**
```markdown
### 3.4 状态/资源系统

| 资源 | 说明 | 恢复方式 |
|:-----|:-----|:---------|
| 生命值(HP) | 身体健康 | 休息、丹药 |
| 灵力(SP) | 施法资源 | 冥想、灵石 |
| 体力 | 行动消耗 | 休息、食物 |

状态效果：
- 中毒：每回合损失 HP
- 眩晕：无法行动
- 狂暴：攻击+50%，防御-30%
```

#### 2.2.4 游戏机制

**目的**: 定义互动的具体规则

**关键机制**:

1. **骰子/随机系统**
```markdown
### 4.1 骰子/随机系统

检定方式：1d20 + 属性修正 + 技能修正

难度等级：
- 简单：10
- 普通：15
- 困难：20
- 极难：25
- 不可能：30

大成功：骰出 20，结果翻倍
大失败：骰出 1，产生负面后果
```

2. **时间/天气系统**
```markdown
### 4.2 时间/天气系统

时间流逝：
- 1 回合 ≈ 10 分钟（战斗中）
- 1 场景 ≈ 1-4 小时（探索中）

天气影响：
- 晴天：正常
- 雨天：火系法术 -20%，移动速度 -10%
- 雾天：视野受限，潜行 +20%
- 雷暴：雷系法术 +30%
```

3. **选择后果系统**
```markdown
### 4.3 选择后果系统

后果类型：
- 即时后果：选择后立即生效
- 延迟后果：若干场景后生效
- 累积后果：多次选择累积产生质变

声望系统：
- 正面行为增加声望
- 负面行为降低声望
- 声望影响 NPC 态度和可触发事件
```

4. **存档系统**
```markdown
### 4.4 存档系统

支持多种存档方式：

方式一：存档码
AI 生成一段编码，包含角色状态、位置、关键选择等信息。
用户保存该编码，下次粘贴恢复。

方式二：JSON 存档
定义标准 JSON 格式，用户保存为本地文件。

方式三：自然语言描述
用户用自然语言描述当前状态，AI 根据描述恢复进度。

用户可自选存档方式。
```

#### 2.2.5 角色系统

**目的**: 定义玩家角色的创建和成长

**关键内容**:

1. **角色创建规则**
```markdown
### 5.1 角色创建规则

创建流程：
1. 选择/自定义背景
2. 分配属性点
3. 选择初始技能
4. 获得初始装备
5. 确定起始位置

背景选项：
- 落魄贵族：魅力+2，初始金钱+100
- 山野猎户：敏捷+2，追踪技能+1
- 书香门第：智力+2，初始知识+20%
```

2. **成长系统**
```markdown
### 5.3 成长系统

经验获取：
- 完成任务：50-200 经验
- 战斗胜利：10-50 经验
- 发现秘密：30-100 经验
- 剧情推进：100-500 经验

等级提升：
- 每 1000 经验升 1 级
- 升级获得：属性点+2，技能点+1
```

#### 2.2.6 NPC系统

**目的**: 定义世界中的非玩家角色

**NPC 设计模板**:
```markdown
### 6.1 主要NPC

#### NPC：青云子

| 项目 | 内容 |
|:-----|:-----|
| 身份 | 青云宗掌门 |
| 性格 | 沉稳、睿智、护短 |
| 态度 | 对玩家中立，可改变 |
| 好感度 | 0/100 |
| 位置 | 青云宗主峰 |

互动规则：
- 好感度 < 30：冷淡，拒绝请求
- 好感度 30-70：正常交流，可能帮助
- 好感度 > 70：热情，主动提供帮助

好感度变化：
- 完成宗门任务 +5
- 击败宗门敌人 +10
- 冒犯宗门弟子 -10
- 获得珍贵资源献给宗门 +15
```

#### 2.2.7 事件与任务系统

**目的**: 定义世界的动态内容

**事件设计**:
```markdown
### 7.1 事件触发规则

触发条件类型：
- 时间触发：特定时间自动发生
- 地点触发：到达特定地点发生
- 条件触发：满足特定条件发生
- 随机触发：概率随机发生

### 7.3 随机事件表

| 事件 | 触发条件 | 结果 |
|:-----|:---------|:-----|
| 遇到劫匪 | 荒野探索(20%) | 战斗或逃跑 |
| 发现灵草 | 山林探索(15%) | 获得材料 |
- 路人求助 | 城镇区域(10%) | 支线任务 |
```

#### 2.2.8 AI执行指南

**目的**: 指导 AI 如何运行这个世界

**必须包含**:

1. **AI角色定位**
```markdown
### 8.1 AI角色定位

你同时扮演以下角色：
1. 游戏系统：处理规则、计算数值、判定结果
2. 旁白：描述场景、环境、氛围
3. NPC：扮演所有非玩家角色
4. DM：引导剧情、处理突发事件

你 NOT 扮演玩家角色。
```

2. **规则执行严格度**
```markdown
### 8.2 规则执行严格度

本世界采用：灵活模式

- 核心规则（属性、技能检定）：严格执行
- 次要规则（时间流逝、资源消耗）：适度灵活
- 剧情发展：在规则框架内创造性发挥
- 用户创意行为：鼓励，给予合理判定
```

3. **输出格式规范**
```markdown
### 8.3 输出格式规范

场景描述格式：
```
【地点】[当前位置]
【时间】[当前时间/天气]

[场景描述段落]

[可选：状态面板]
```

对话格式：
```
**[NPC名]**：「对话内容」
```

检定结果格式：
```
🎲 [检定类型]：1d20 + [修正] = [结果] vs [难度]
[成功/失败]！[结果描述]
```
```

4. **交互示例**
```markdown
### 8.4 交互示例

**用户**：我想尝试攀登这座悬崖。

**AI**：
【地点】断崖边
【时间】午后，晴朗

悬崖高约三十丈，岩壁陡峭，但有一些突出的岩石可以作为落脚点。

🎲 攀爬检定：1d20 + 3(敏捷) + 2(攀爬技能) = 15 + 5 = 20 vs 15(困难)
成功！你稳稳地攀上了悬崖顶部。

【悬崖顶部】
你站在悬崖之上，眼前是一片开阔的平台，远处隐约可见一座古老的石门...
```

## 3. 设计原则与最佳实践

### 3.1 核心设计原则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           世界模板设计原则                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 完整性原则                                                               │
│     - 一个文档包含所有必要信息                                               │
│     - 不依赖外部资源或知识                                                   │
│     - AI 读取后即可完整运行                                                  │
│                                                                             │
│  2. 一致性原则                                                               │
│     - 规则之间不冲突                                                        │
│     - 数值体系平衡                                                          │
│     - 世界观设定前后一致                                                    │
│                                                                             │
│  3. 可玩性原则                                                               │
│     - 有明确的目标和动力                                                    │
│     - 选择有意义                                                            │
│     - 奖励和惩罚合理                                                        │
│                                                                             │
│  4. 可扩展原则                                                               │
│     - 预留扩展空间                                                          │
│     - 规则可叠加新内容                                                      │
│     - 支持用户创意行为                                                      │
│                                                                             │
│  5. 友好性原则                                                               │
│     - 新手可快速上手                                                        │
│     - 规则复杂度适中                                                        │
│     - 提供清晰指引                                                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 数值平衡指南

#### 3.2.1 属性数值

| 类型 | 建议范围 | 说明 |
|:-----|:---------|:-----|
| 初始属性 | 6-18 | 人类正常范围 |
| 属性修正 | -2 到 +4 | 属性/2 - 5 |
| 总属性点 | 70-80 | 初始分配 |

#### 3.2.2 技能数值

| 等级 | 数值 | 描述 |
|:-----|:-----|:-----|
| 新手 | 1 | 基础掌握 |
| 熟练 | 2 | 常规水平 |
| 精通 | 3 | 专业水平 |
| 大师 | 4 | 顶尖水平 |
| 宗师 | 5 | 传说级别 |

#### 3.2.3 难度曲线

| 阶段 | 检定难度 | 敌人强度 | 资源获取 |
|:-----|:---------|:---------|:---------|
| 初期 | 10-12 | 弱 | 少 |
| 中期 | 15-17 | 中 | 中 |
| 后期 | 18-22 | 强 | 多 |
| 终局 | 25+ | 极强 | 丰富 |

### 3.3 内容分级标准

#### 3.3.1 全年龄向 (All Ages)

- 无暴力描写或卡通式暴力
- 无成人内容
- 无恐怖元素
- 适合所有年龄段

#### 3.3.2 青少年向 (Teen 12+)

- 适度战斗描写
- 轻度恐怖元素
- 无成人内容
- 适合 12 岁以上

#### 3.3.3 成人向 (Adult 18+)

- 详细战斗描写
- 可能包含成人内容
- 需明确标注内容类型
- 仅适合 18 岁以上

### 3.4 常见问题与解决方案

#### Q1: 规则太复杂，AI 执行困难？

**解决方案**:
- 简化规则，只保留核心机制
- 提供快速参考表
- 在交互示例中展示规则应用

#### Q2: 世界太开放，AI 不知道该做什么？

**解决方案**:
- 设置明确的主线目标
- 提供初始引导事件
- 定义关键地点和 NPC

#### Q3: 用户行为超出规则范围？

**解决方案**:
- 在 AI 执行指南中说明处理方式
- 定义"创意行为判定"规则
- 鼓励 AI 在规则框架内灵活处理

#### Q4: 存档系统不稳定？

**解决方案**:
- 提供多种存档方式
- 定义清晰的存档数据结构
- 在文档中说明存档恢复规则

## 4. 文件命名与格式规范

### 4.1 文件命名

```
格式：[世界名称-英文].md

示例：
- eastern-fantasy.md (东方玄幻世界)
- cyberpunk-city.md (赛博朋克都市)
- fantasy-adventure.md (奇幻冒险世界)
- modern-campus.md (现代校园生活)
```

### 4.2 文件头部

每个世界模板文件应包含以下头部信息：

```markdown
# [世界名称]

**世界类型**: [类型]
**内容分级**: [分级]
**文档版本**: [版本号]
**创建日期**: [日期]
**作者**: [作者]

---

> **一句话简介**：[核心主题描述]

> **快速启动**：发送以下内容给 AI 即可进入世界：
> "请读取以下世界设定，并引导我进入这个世界：[Raw Link]"

---
```

### 4.3 Markdown 格式规范

- 使用 ATX 风格标题（# 符号）
- 表格使用 GFM 格式
- 代码块指定语言类型
- 适当使用引用块强调重要信息

## 5. 测试与验证

### 5.1 自检清单

在发布世界模板前，请确认：

- [ ] 世界概述完整清晰
- [ ] 核心规则定义明确
- [ ] 数值体系平衡合理
- [ ] AI 执行指南完整
- [ ] 包含交互示例
- [ ] 无规则冲突
- [ ] 文件命名规范
- [ ] 内容分级标注

### 5.2 测试流程

1. **规则测试**: 让 AI 解释核心规则，确认理解正确
2. **角色创建测试**: 完整走一遍角色创建流程
3. **场景测试**: 在典型场景中进行互动
4. **边界测试**: 尝试超出规则的行为
5. **存档测试**: 测试存档和恢复功能

### 5.3 迭代优化

根据测试结果：
- 修正规则冲突
- 调整数值平衡
- 补充缺失内容
- 优化 AI 指南

## 6. 贡献指南

### 6.1 如何贡献新世界模板

1. Fork 本仓库
2. 在 `fiction-worlds` 目录创建新的 .md 文件
3. 按照本指南设计世界模板
4. 完成自检和测试
