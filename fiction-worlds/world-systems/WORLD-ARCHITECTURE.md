**文档版本**: 2.0

## 设计哲学

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           核心设计理念                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   万物皆对象                                                                 │
│   ─────────                                                                 │
│   世界中的一切——山川、灵脉、丹药、功法、势力、事件——都是可定义的对象。          │
│   每个对象有属性、有行为、有关系、有生命周期。                                 │
│                                                                             │
│   万物有数值                                                                 │
│   ─────────                                                                 │
│   一切皆可量化。灵气浓度、资质、气运、仇恨值、势力声望……                       │
│   数值决定行为，数值驱动演化。                                               │
│                                                                             │
│   万物可演化                                                                 │
│   ─────────                                                                 │
│   世界不是静态的背景板，而是动态演化的生态系统。                               │
│   灵气枯竭导致宗门衰落，资源争夺引发战争，因果循环产生事件链。                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## AI执行要点

### 1. 数据驱动原则

所有判定必须基于数值，而非主观判断：
- ✅ 正确：根据灵气浓度45，判定该区域可生长4阶灵草
- ❌ 错误：根据感觉，判定该区域有一些灵草

### 2. 演化优先原则

世界状态会随时间和玩家行为变化：
- 灵气被过度开采 → 浓度下降 → 宗门迁移 → 区域衰落
- 玩家击杀大量妖兽 → 妖兽密度下降 → 灵草生长增加

### 3. 因果联动原则

每个行为都有后果，形成因果链：
- 杀人 → 仇恨值增加 → 可能触发追杀/复仇事件
- 救人 → 因果值增加 → 可能触发报恩事件

### 4. 状态持久化

所有状态变化需要记录：
- 区域状态变化
- 势力关系变化
- NPC好感度变化
- 因果值变化
