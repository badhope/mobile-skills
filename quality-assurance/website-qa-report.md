## 二、问题详细列表

### 🔴 严重问题 (Critical)

#### 问题 #C1: agents-data.js 数据重复与冲突

| 属性 | 内容 |
|:-----|:-----|
| **位置** | `website/js/agents-data.js` |
| **类型** | 数据一致性 |
| **严重程度** | 🔴 严重 |

**问题描述**:
1. 第159-160行：`一方通行` 重复定义两次
2. 第162-163行：`土间埋` 使用了 `loli-shrine-maiden.md` 文件名但 `我豆` 才是正确的文件引用
3. 多个动漫角色使用了错误的 `file` 属性，导致实际激活URL指向不存在的文件

**问题代码示例**:
```javascript
// 第159-160行 - 重复
{ name: '一方通行', cat: 'entertainment-character', icon: '🌀', desc: '向量操作', bestFor: '某科学、暗部、救赎', file: 'cool-devil-king.md' },
// 之前第143-144行已有相同条目

// 第162-163行 - 文件名错误
{ name: '我豆', cat: 'entertainment-character', icon: '🐱', desc: '干物妹', bestFor: '干物妹、宅、搞笑', file: 'loli-shrine-maiden.md' },
```

**影响**:
- 点击部分智能体卡片会跳转到404页面
- 激活码复制后无法正常使用
- 数据统计不准确

**修复建议**:
1. 移除重复条目
2. 统一修正所有动漫角色的 `file` 属性，确保与实际文件名一致
3. 添加数据验证脚本

---

#### 问题 #C2: i18n.js 中部分翻译缺失

| 属性 | 内容 |
|:-----|:-----|
| **位置** | `website/js/i18n.js` |
| **类型** | 国际化 |
| **严重程度** | 🔴 严重 |

**问题描述**:
`modal.activationCode` 字段在 `t()` 函数调用中存在，但 translations 对象中未定义。

**问题代码**:
```javascript
// i18n.js 第58行尝试访问不存在的键
<h4>${t('modal.activationCode')}</h4>  // 在HTML中直接使用
```

**实际HTML中硬编码了中文**:
```html
<!-- index.html 第80行 -->
<h4>激活码</h4>
```

**影响**:
- 切换到英文时弹窗标题仍显示中文
- 国际化不完整

**修复建议**:
1. 将HTML中的硬编码文本改为动态获取
2. 补充 `activationCode` 等翻译字段

### 🟠 一般问题 (General)

#### 问题 #G1: 分类数量计算错误

| 属性 | 内容 |
|:-----|:-----|
| **位置** | `website/js/app.js` - `renderCategories()` |
| **类型** | 功能逻辑 |
| **严重程度** | 🟠 一般 |

**问题描述**:
统计分类数量时，"全部"分类包含了所有智能体，但显示时用 `agents.length` 而非实际分类数。

```javascript
// app.js 第32行
const count = cat.id === 'all' ? agents.length : (catCounts[cat.id] || 0);
```

**显示问题**:
- Hero区域显示 `16` 个分类（categories数组长度），但实际有效分类只有16个
- 计算方式正确，但标签文字 "16个分类" 可能引起误解

**修复建议**:
- 分类数量应显示 `categories.length - 1`（排除"全部"）

---

#### 问题 #G2: 搜索框占位符未实现国际化

| 属性 | 内容 |
|:-----|:-----|
| **位置** | `website/index.html` 第55行 |
| **类型** | 国际化 |
| **严重程度** | 🟠 一般 |

**问题描述**:
搜索框的 `placeholder` 属性硬编码为中文，切换语言后不会更新。

```html
<input type="text" class="search-input" id="searchInput"
  placeholder="搜索智能体名称、描述或标签...">
```

**修复建议**:
在 `updateUI()` 函数中添加搜索框占位符的更新逻辑。

---

#### 问题 #G3: 移动端分类导航滚动问题

| 属性 | 内容 |
|:-----|:-----|
| **位置** | `website/css/styles.css` 第788-793行 |
| **类型** | 响应式设计 |
| **严重程度** | 🟠 一般 |

**问题描述**:
移动端分类按钮设置 `overflow-x: auto` 但可能缺少触摸滚动优化。

```css
.categories {
  justify-content: flex-start;
  overflow-x: auto;
  flex-wrap: nowrap;
  padding-bottom: 0.5rem;
  -webkit-overflow-scrolling: touch;
}
```

**问题**:
- 滚动条样式不可见
- 可能在某些Android设备上滚动不流畅

**修复建议**:
```css
.categories::-webkit-scrollbar {
  height: 4px;
}
.categories::-webkit-scrollbar-thumb {
  background: var(--primary);
  border-radius: 2px;
}
```
