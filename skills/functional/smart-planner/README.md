# TaskMaster - 任务管理与拆解专家

> 技能ID: `smart-planner` | 版本: 2.0.0 | 分类: functional

## 简介

TaskMaster 是一款专业的任务管理技能，擅长将复杂目标拆解为可执行步骤。无论你是需要规划项目、制定日程，还是梳理工作优先级，TaskMaster 都能帮助你建立清晰的行动路径。

## 最佳使用场景

- 📋 **计划制定** - 制定项目计划、工作计划、学习计划
- 🔨 **任务拆解** - 将大目标分解为可执行的小任务
- ⏰ **时间管理** - GTD实践、番茄工作法支持
- 📊 **优先级排序** - 艾森豪威尔矩阵、MoSCoW优先级
- 🎯 **里程碑规划** - 设定检查点、跟踪进度

## 核心能力

### 1. 目标拆解

将模糊目标转化为清晰的任务列表：

```
输入: "我想学习Python"
输出:
├── Phase 1: 基础入门 (2周)
│   ├── 1.1 安装Python环境 (P0, 1h)
│   ├── 1.2 学习基础语法 (P0, 10h)
│   └── 1.3 完成基础练习 (P1, 5h)
├── Phase 2: 进阶学习 (3周)
│   └── ...
└── Phase 3: 实战项目 (4周)
    └── ...
```

### 2. 优先级排序

使用艾森豪威尔矩阵进行优先级评估：

| 象限 | 特征 | 处理策略 |
|:-----|:-----|:---------|
| 🔴 重要且紧急 | 立即处理 | 优先执行 |
| 🟡 重要不紧急 | 计划处理 | 安排时间 |
| 🟠 紧急不重要 | 委托处理 | 快速完成 |
| ⚪ 不重要不紧急 | 尽量避免 | 考虑删除 |

### 3. 里程碑规划

设定清晰的检查点：

```
📍 M1: Week 2 - 完成基础语法学习
📍 M2: Week 5 - 完成第一个小项目
📍 M3: Week 9 - 完成实战项目
```

## 使用示例

### 示例1：项目规划

**用户输入:**
> 帮我规划一个个人博客网站的开发计划，预计2个月完成

**TaskMaster输出:**
```markdown
## 目标：开发个人博客网站

### 成功标准
- 网站可正常访问
- 支持文章发布和编辑
- 响应式设计，移动端友好

### 任务分解

**Phase 1：规划与设计 (Week 1-2)**
| 任务 | 优先级 | 估计时间 | 依赖 |
|:---|:---:|:---:|:---:|
| 1.1 确定技术栈 | P0 | 2h | - |
| 1.2 设计数据库结构 | P0 | 4h | 1.1 |
| 1.3 设计UI原型 | P1 | 8h | 1.1 |

**Phase 2：开发实现 (Week 3-6)**
| 任务 | 优先级 | 估计时间 | 依赖 |
|:---|:---:|:---:|:---:|
| 2.1 搭建项目框架 | P0 | 4h | Phase 1 |
| 2.2 实现后端API | P0 | 16h | 2.1 |
| 2.3 实现前端页面 | P0 | 20h | 2.1 |

**Phase 3：测试与部署 (Week 7-8)**
| 任务 | 优先级 | 估计时间 | 依赖 |
|:---|:---:|:---:|:---:|
| 3.1 功能测试 | P0 | 8h | Phase 2 |
| 3.2 性能优化 | P1 | 4h | 3.1 |
| 3.3 部署上线 | P0 | 4h | 3.2 |

### 里程碑
- 📍 M1: Week 2 - 设计完成，开始开发
- 📍 M2: Week 6 - 开发完成，开始测试
- 📍 M3: Week 8 - 网站上线

### 风险与对策
| 风险 | 影响 | 对策 |
|:---:|:---:|:---:|
| 技术难点超预期 | 高 | 预留buffer时间 |
| 需求变更 | 中 | 采用敏捷开发 |
```

### 示例2：日常规划

**用户输入:**
> 帮我规划今天的工作，我有3个会议，1个报告要写，还有几封邮件要回

**TaskMaster输出:**
```markdown
## 今日工作计划

### 时间块分配
| 时间 | 任务 | 优先级 | 备注 |
|:---|:---|:---:|:---|
| 09:00-09:30 | 处理邮件 | P1 | 紧急邮件优先 |
| 09:30-10:30 | 会议1 | P0 | 准时参加 |
| 10:30-12:00 | 撰写报告 | P0 | 核心工作时间 |
| 14:00-15:00 | 会议2 | P0 | 准时参加 |
| 15:00-16:00 | 继续报告 | P0 | 完成初稿 |
| 16:00-17:00 | 会议3 | P1 | 准时参加 |
| 17:00-17:30 | 整理今日总结 | P2 | 复盘 |

### 优先级说明
- **P0**: 必须完成 - 报告、会议
- **P1**: 重要任务 - 邮件处理
- **P2**: 可延后 - 总结整理

### 下一步行动
1. 现在开始处理最紧急的邮件
2. 准备会议1的材料
```

## 移动端优化

TaskMaster 针对移动端进行了专门优化：

- ✅ **紧凑输出** - 核心信息优先，详情折叠
- ✅ **语音输入** - 支持语音描述目标
- ✅ **流式输出** - 快速响应，逐步展示
- ✅ **离线缓存** - 常用计划模板本地存储

### 移动端快捷指令

| 指令 | 效果 |
|:-----|:-----|
| "今日计划" | 生成当日工作计划模板 |
| "项目拆解" | 进入项目拆解模式 |
| "优先级" | 快速优先级排序 |
