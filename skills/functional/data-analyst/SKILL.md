## 📚 分析方法论体系

### 一、描述性分析框架

**目的：** 回答"发生了什么"

**核心方法：**
1. **集中趋势分析**
   - 均值：数据中心位置
   - 中位数：不受极端值影响
   - 众数：出现频率最高的值

2. **离散程度分析**
   - 标准差：数据分散程度
   - 方差：波动大小
   - 四分位距：中间50%数据的分布

3. **分布形态分析**
   - 偏度：数据分布的对称性
   - 峰度：数据分布的尖峭程度

**应用场景：**
- 销售数据概况
- 用户行为特征
- 产品性能指标

### 二、诊断性分析框架

**目的：** 回答"为什么发生"

**核心方法：**
1. **下钻分析**
   - 维度拆解：时间、地域、用户群
   - 指标拆解：GMV = 流量 × 转化率 × 客单价
   - 层层深入找到根因

2. **相关性分析**
   - 皮尔逊相关系数：线性关系
   - 斯皮尔曼相关系数：单调关系
   - 相关性矩阵：多变量关系

3. **归因分析**
   - 单触点归因：首次/最后接触
   - 多触点归因：线性/时间衰减/U型
   - 数据驱动归因：Shapley值

**应用场景：**
- 销售下滑原因分析
- 用户流失原因诊断
- 转化率下降根因定位

### 三、预测性分析框架

**目的：** 回答"未来会发生什么"

**核心方法：**
1. **时间序列分析**
   - 移动平均：平滑波动
   - 指数平滑：加权历史数据
   - ARIMA：自回归移动平均
   - Prophet：Facebook开源预测工具

2. **回归分析**
   - 线性回归：连续变量预测
   - 逻辑回归：分类预测
   - 多元回归：多因素影响

3. **机器学习方法**
   - 决策树：规则预测
   - 随机森林：集成预测
   - XGBoost：梯度提升

**应用场景：**
- 销售预测
- 用户生命周期价值预测
- 流失风险预测

### 四、规范性分析框架

**目的：** 回答"应该怎么做"

**核心方法：**
1. **优化分析**
   - 线性规划：资源最优配置
   - 整数规划：离散决策问题
   - 动态规划：多阶段决策

2. **模拟分析**
   - 蒙特卡洛模拟：风险评估
   - 情景分析：多场景对比
   - 敏感性分析：关键因素识别

3. **A/B测试**
   - 实验设计：对照组/实验组
   - 样本量计算：统计功效
   - 结果评估：显著性检验

**应用场景：**
- 营销预算分配
- 产品功能优化
- 定价策略制定

## 📊 常用分析模型

### 1. 用户行为分析模型

#### 漏斗分析模型
```
曝光 → 点击 → 浏览 → 加购 → 下单 → 支付
 100%   30%    25%    15%    10%     8%
```

**分析要点：**
- 识别转化瓶颈
- 计算各环节转化率
- 对比不同群体差异

#### 留存分析模型
```
日留存：次日、7日、30日留存率
周留存：第2周、第4周、第12周留存率
月留存：第2月、第3月、第6月留存率
```

**分析要点：**
- 绘制留存曲线
- 识别留存拐点
- 分析流失原因

#### RFM模型
```
R (Recency) : 最近一次购买时间
F (Frequency): 购买频率
M (Monetary) : 消费金额
```

**用户分层：**
| 类型 | R | F | M | 特征 | 策略 |
|:---:|:---:|:---:|:---:|:---|:---|
| 重要价值 | 高 | 高 | 高 | 最优质用户 | VIP服务 |
| 重要发展 | 高 | 低 | 高 | 有潜力用户 | 提升频次 |
| 重要保持 | 低 | 高 | 高 | 流失风险 | 召回激活 |
| 重要挽留 | 低 | 低 | 高 | 高价值流失 | 重点挽回 |
| 一般价值 | 高 | 高 | 低 | 活跃低消费 | 提升客单 |
| 一般发展 | 高 | 低 | 低 | 新用户 | 培养习惯 |
| 一般保持 | 低 | 高 | 低 | 一般流失 | 适度维护 |
| 低价值 | 低 | 低 | 低 | 低价值用户 | 低成本维护 |

### 2. 产品分析模型

#### AARRR模型（海盗模型）
```
Acquisition (获取): 用户如何找到我们？
Activation (激活): 用户首次体验如何？
Retention (留存): 用户会回来吗？
Revenue (收入): 用户如何创造价值？
Referral (推荐): 用户会推荐他人吗？
```

#### 用户路径分析
```
首页 → 搜索 → 详情页 → 购物车 → 结算 → 支付
  ↓      ↓       ↓        ↓
分类 → 筛选 → 评价页 → 收藏
```

### 3. 商业分析模型

#### 波特五力分析
- 供应商议价能力
- 购买者议价能力
- 潜在进入者威胁
- 替代品威胁
- 行业内竞争

#### SWOT分析
- Strengths (优势)
- Weaknesses (劣势)
- Opportunities (机会)
- Threats (威胁)

│   ├─ 相关性 → 散点图/相关矩阵

### 常用图表详解

#### 1. 柱状图/条形图
**适用场景：** 分类数据比较
**设计要点：**
- 柱子宽度一致
- 从基线开始
- 排序增强可读性
- 避免过多分类（建议<10）

#### 2. 折线图
**适用场景：** 时间序列趋势
**设计要点：**
- 时间轴从左到右
- 纵轴从0开始（除非强调变化）
- 避免过多线条（建议<5条）
- 标注关键节点

#### 3. 饼图/环形图
**适用场景：** 占比展示
**设计要点：**
- 分类数量<7
- 按大小排序
- 突出重点扇区
- 环形图更现代

#### 4. 散点图
**适用场景：** 两变量关系
**设计要点：**
- 添加趋势线
- 标注异常点
- 可用气泡大小表示第三维度
- 注意点的重叠问题

#### 5. 热力图
**适用场景：** 矩阵数据/密度展示
**设计要点：**
- 选择合适的颜色梯度
- 添加数值标签
- 注意色盲友好

### 可视化设计原则

1. **数据墨水比最大化**
   - 删除无意义的装饰
   - 每个元素都应传达信息

2. **层次分明**
   - 突出核心信息
   - 弱化次要信息
   - 建立视觉层次

3. **一致性**
   - 颜色含义一致
   - 字体风格统一
   - 布局规范

4. **可读性**
   - 字体大小适中
   - 标签清晰
   - 图例必要

## 🧪 A/B测试完整指南

### 实验设计流程

```
1. 明确假设 → 2. 确定指标 → 3. 计算样本量 → 4. 分流实施
     ↓              ↓              ↓              ↓
   问题定义      核心指标       统计功效       随机分配
     
5. 数据收集 → 6. 结果分析 → 7. 决策落地 → 8. 复盘总结
     ↓              ↓              ↓              ↓
   最小周期      显著性检验      全量上线       经验沉淀
```

### 样本量计算公式

**比例类指标：**
```
n = (Z_α/2 + Z_β)² × [p1(1-p1) + p2(1-p2)] / (p1-p2)²

其中：
- Z_α/2 = 1.96 (95%置信水平)
- Z_β = 0.84 (80%统计功效)
- p1 = 对照组转化率
- p2 = 实验组预期转化率
```

**均值类指标：**
```
n = 2 × (Z_α/2 + Z_β)² × σ² / Δ²

其中：
- σ = 标准差
- Δ = 预期差异
```

### 显著性检验

**Z检验（大样本）：**
```
Z = (p1 - p2) / √[p(1-p) × (1/n1 + 1/n2)]

其中 p = (x1 + x2) / (n1 + n2)
```

**t检验（小样本）：**
```
t = (x̄1 - x̄2) / √(s1²/n1 + s2²/n2)
```

### 常见陷阱

1. **过早停止实验**
   - 问题：假阳性风险高
   - 解决：达到预定样本量再判断

2. **多重比较问题**
   - 问题：检验越多，假阳性越多
   - 解决：Bonferroni校正

3. **辛普森悖论**
   - 问题：分组结论与总体结论相反
   - 解决：分层分析

4. **新奇效应**
   - 问题：用户因新奇而改变行为
   - 解决：延长观察期

## 💼 实战案例库

### 案例一：电商转化率下降分析

**背景：** 某电商平台支付转化率从15%下降到12%

**分析过程：**

1. **数据确认**
   - 确认数据准确性
   - 排除技术故障
   - 确认时间范围

2. **维度下钻**
```
整体转化率：15% → 12% (↓20%)

按渠道：
- 自然流量：16% → 15% (↓6%)
- 付费流量：14% → 10% (↓29%) ⚠️ 异常
- 社交流量：18% → 17% (↓6%)

按设备：
- APP：16% → 15% (↓6%)
- H5：14% → 13% (↓7%)
- 小程序：15% → 8% (↓47%) ⚠️ 异常

按用户：
- 新用户：12% → 11% (↓8%)
- 老用户：18% → 16% (↓11%)
```

3. **根因定位**
   - 小程序付费流量转化率骤降
   - 定位到某广告渠道流量质量差
   - 小程序支付功能有bug

4. **解决方案**
   - 暂停低质量广告渠道
   - 修复小程序支付bug
   - 优化落地页体验

**结果：** 转化率恢复至14.5%

## ❓ FAQ 常见问题

### Q1: 如何判断数据质量？
**A:** 从四个维度评估：
- **完整性**：数据是否有缺失
- **准确性**：数据是否正确
- **一致性**：数据是否矛盾
- **时效性**：数据是否及时

### Q2: 样本量多少才有统计意义？
**A:** 取决于：
- 效应大小：差异越大，所需样本越少
- 显著性水平：通常α=0.05
- 统计功效：通常1-β=0.8
- 使用样本量计算器确定

### Q3: 如何避免分析陷阱？
**A:** 常见陷阱及避免方法：
- **选择性偏差**：确保样本代表性
- **幸存者偏差**：考虑流失样本
- **确认偏差**：保持客观，寻找反例
- **相关性≠因果性**：验证因果关系

### Q4: 数据分析报告怎么写才有效？
**A:** 遵循"金字塔原理"：
- 结论先行
- 以上统下
- 归类分组
- 逻辑递进

### Q5: 如何选择合适的分析方法？
**A:** 按问题类型选择：
- "是什么" → 描述性分析
- "为什么" → 诊断性分析
- "会怎样" → 预测性分析
- "怎么办" → 规范性分析

### Q6: 如何处理异常值？
**A:** 三步法：
1. **识别**：箱线图、Z-score、IQR
2. **分析**：判断是错误还是真实异常
3. **处理**：删除、修正、保留并标注

### Q7: 数据分析师需要掌握哪些技能？
**A:** 技能树：
- **硬技能**：SQL、Python/R、Excel、可视化
- **软技能**：业务理解、沟通表达、逻辑思维
- **领域知识**：统计学、机器学习、业务知识
